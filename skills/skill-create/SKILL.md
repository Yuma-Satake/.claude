# skill-create

skill の新規作成・改善を行う際は、まず Skill ツールで `skill-creator:skill-creator`（Anthropic 公式プラグイン）をロードし、その手順（ドラフト作成 → テスト実行 → 評価 → 改善のループ）に従う。以下は公式の手順に対する、このユーザー環境での個別ナレッジ。

## 設計方針

- 1スキル = 1責務とする。スキルの1ステップが独立した関心事に育って肥大化してきたら、別スキルに切り出し、呼び出し元には「Skill ツールで必ず呼び出す」と明記した参照だけを残す（例: commit スキルから knowledge-review スキルを呼ぶ構成）
- スキル名は目的が名前から分かる名詞ベースで付ける（例: knowledge-review、skill-create）

## 評価・改善の進め方

- スキルの改善は評価駆動で行う。改善したい挙動と防ぎたい挙動をアサーションとして明文化し、旧版（スナップショット）と改善版をサブエージェントで並列実行して比較する
- 会話全体を対象とするスキル（振り返り・レビュー系）は、シグナルを埋め込んだ模擬会話トランスクリプトと模擬設定ディレクトリ（mock の rules・skills・プロジェクト）を用意すれば定量評価できる。ユーザー確認が必要なステップは、サブエージェントには提案書ファイルへの書き出しで代替させる
- 評価用ワークスペースは scratchpad に置く。公式 skill-creator は「スキルディレクトリの隣に置く」と指示するが、~/.claude は git リポジトリのため評価成果物で汚さない
- 評価のサブエージェントは、そのスキルが実際に動くモデルで実行する（例: model: sonnet のスキルなら評価も sonnet で行う）
- サブエージェントの並列駆動は agent-use スキルの原則に従う（成果物ファイル名に analysis・report 系を避ける点に注意）
