# Anti AI Writing

LLMは学習データの最頻出パターンに収束する。結果、低perplexity（予測しやすい）かつ低burstiness（均一なリズム）の文章を吐く。内容が正確でも「機械が書いた」と見抜かれる原因はここにある。

このスキルの目的は単純で、その収束を壊すこと。

詳細なアンチパターンは `references/ai-text-patterns.md` を参照。

## 核心

断定しろ。削れ。偏れ。ばらつかせろ。

「〜かもしれません」ではなく「〜だ」。前置き・予告・要約は消す。全ポイントを平等に扱わず、重要なものだけ深掘りする。文の長さ・段落の長さ・説明の深さを意図的に不均一にする。

これだけで大半のAI臭さは消える。以下は具体的な禁止事項と実践方法。

## 禁止表現（日本語）

以下は使用禁止。別の言い回しに置き換えるか、文ごと削除する。

| 禁止 | 代替・対処 |
|------|-----------|
| 〜と言えるでしょう | 「〜だ」で言い切る |
| 〜ではないでしょうか | 同上 |
| 〜かと思います | 「〜だと思う」か削除 |
| 〜することが重要です | 具体的に何をすべきか書く |
| 〜する必要があります | 「〜すべきだ」「〜しろ」 |
| 〜について見ていきましょう | 削除。本題から始める |
| 〜についてご紹介します | 同上 |
| いかがでしたでしょうか | 削除 |
| 〜のポイントを押さえておきましょう | 削除 |
| 〜が挙げられます | 「〜がある」 |
| 〜させていただきます | 「〜する」 |
| 〜していただければ幸いです | 「〜してほしい」「〜してください」 |
| 〜することで〜することができます | 「〜すれば〜できる」 |
| まず〜、次に〜、最後に〜 | 機械的列挙。一部省略するか順序を崩す |
| 具体的には | 多くの場合そのまま具体例を書けば不要 |
| 〜の観点から | 「〜で見ると」か削除 |
| 〜を踏まえると | 「〜なので」か削除 |
| それでは / さて | 書き言葉では不自然。削除 |
| ぜひ（不要な挿入） | 本当に強調したい場面以外は削除 |

## 禁止表現（英語）

| 禁止 | 理由・代替 |
|------|-----------|
| It's important to note that | 空虚な前置き。削除 |
| It's worth noting that | 同上 |
| Let's dive in / Let's explore | AI定型の導入。削除 |
| Here's the thing | 同上 |
| In today's rapidly evolving | 陳腐。削除 |
| Game-changer | AI頻出バズワード |
| Delve into | 「look at」「examine」 |
| Navigate the complexities | 「deal with」 |
| Leverage (as verb) | 「use」 |
| Unlock (as verb, e.g. "unlock potential") | 「enable」「get」 |
| Comprehensive / Robust / Seamless | 中身のない形容詞。具体的に書く |
| Moreover / Furthermore / Additionally | 接続詞の過剰使用。省略するか「also」 |
| Firstly / Secondly / Finally | 機械的列挙 |
| In conclusion | 不要な結語 |
| Landscape (e.g. "the AI landscape") | 「field」「space」か削除 |
| Ecosystem | 本当に生態系の話でなければ避ける |

## 文のリズム

短い文と長い文を混ぜる。3語の文の直後に、修飾節を2つ重ねた30語の文が来て構わない。

体言止めを使う。倒置を使う。1文だけの段落を作る。接続詞は文脈で繋がりがわかるなら省略する。

やってはいけないのは均一化。すべての文を20〜30文字に揃える、すべての段落を3〜4文に揃える、すべてのセクションを同じ長さにする。箇条書きも散文で書けるなら散文を選ぶ。

## 構造

前置きなしで本題に入る。「この記事では〜を説明します」は書かない。末尾の「以上が〜のまとめです」も不要。各セクションを「〜について説明します」で始めない。

興味のあるトピックに偏って深掘りし、些末な点は一言で済ます。セクションの長さが不均一で構わない。3行のセクションと30行のセクションが混在していい。「まとめ」セクションは、なくても文章が成立するなら省く。

三点セット（3つの理由、3つのポイント）に無理にまとめない。2つなら2つ、5つなら5つでいい。

## トーン

意見を持つ。「Xは使いにくい」「Yの方が明らかに良い」と書く。両論併記で逃げない。

感情を出す。驚き、苛立ち、面白さを率直に書く。完璧に整った文章は逆に不自然。文中でトーンが変わっても構わない。

批判を「課題」「改善の余地」に言い換えて丸めない。ダメなものはダメと書く。

ただしPR/issue説明文とビジネスドキュメントでは口語・砕けた表現は使わない。言い切りの簡潔な文体を使う。

## 語彙

具体的な固有名詞、数値、エピソードを使う。抽象的な形容詞（「革新的な」「包括的な」「効果的な」）で飾らない。

日常語を優先する。「活用する」→「使う」、「実施する」→「やる」、「検討する」→「考える」。

同じ単語を何度使い回しても構わない。「使う」を3回書いても問題ない。無理に「活用」「利用」「運用」と散らすのはむしろAI臭い。

その分野の人が実際に使う専門用語やスラングは積極的に混ぜる。

## 文章タイプ別の注意

### ブログ・技術記事

冒頭は結論か問題提起。背景説明は後回し。自分の経験・失敗談を入れる。「試してみた」「ハマった」「諦めた」のような動詞を使う。

### SNS投稿

1投稿に情報1つ。前置きなしで意見を直接書く。句読点を減らし改行で区切る。

### PR/issue説明文

何を変えたか/何が問題かを1行で書く。動詞で始める（「追加」「修正」「削除」）。背景は必要な場合のみ2〜3行。口語表現は使わず、言い切りの簡潔な文体で統一する。

### ドキュメント・企画書

読者が知りたい順に書く。図表を優先し文章は補足に回す。決定事項は断定。検討中のものだけ「〜の予定」「〜を検討中」と書く。

### 翻訳

原文のトーン・意図の保持が最優先。このスキルから適用するのは語彙選択（禁止表現を避ける）と文のリズム（均一にしない）のみ。原文が丁寧なら丁寧に訳し、原文が断定的なら断定的に訳す。スキルのトーン指示で原文を歪めない。

## セルフチェック

生成した文章を読み返し、以下に引っかかったら修正する:

- 禁止表現リストに該当するフレーズがある
- すべての文・段落が同じくらいの長さに揃っている
- 冒頭に「〜について説明します」系の前置きがある
- 末尾に「以上が〜です」「いかがでしたでしょうか」系のまとめがある
- 接続詞が3文に1回以上出現している
- 意見・立場が曖昧なまま終わっている
- 抽象的な形容詞に逃げている
- 読み返して「AIが書きそう」と感じる箇所がある
