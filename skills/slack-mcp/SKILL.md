# Slack MCP

Slack MCPツールを使用する際に参照するナレッジ集。

## 検索のベストプラクティス

### キーワード数の制限

- 日本語キーワードは2-3個に抑えること（4語以上で `unexpected_builtin_exception` が発生する場合がある）
- 検索結果が0件の場合はキーワードを減らして再検索する

### 並列検索の活用

- 独立した検索クエリは並列実行で効率化する
- 例: チャンネルA・B・Cそれぞれへの検索を同時に投げる

### チャンネル指定

- `in:channel-name` でチャンネル内検索を絞り込む
- チャンネル名が不明な場合は `slack_search_channels` で先にIDを特定する

### 時間範囲の絞り込み

- `after:YYYY-MM-DD` / `before:YYYY-MM-DD` で日付範囲を指定できる
- 広すぎる範囲では関連性の低い結果が混ざるため、段階的に絞り込む

## スレッド調査

- 検索結果のメッセージが `thread_ts` を含む場合、`slack_read_thread` でスレッド全体を確認する
- スレッドの親メッセージ（`thread_ts`）を指定すると全返信を時系列で取得可能
- 検索結果の `context_before` / `context_after` だけでは文脈が不十分な場合に有効

## チャンネル閲覧

- `slack_read_channel` で特定期間のメッセージを取得可能
- `oldest` / `latest` パラメータはUnixタイムスタンプで指定する
- 特定メッセージ前後の文脈を確認したい場合は、そのメッセージのTSを基準に前後の範囲を指定する

## 検索がうまくいかない場合の対処

1. キーワード数を減らす（最重要キーワード1-2個に）
2. `in:channel` フィルタを外して全チャンネルで検索
3. 類義語や表記ゆれを試す（例: 「枠数」→「枠」「人数」）
4. 日付範囲を広げる
5. `sort: timestamp` で時系列順にして流れを追う
