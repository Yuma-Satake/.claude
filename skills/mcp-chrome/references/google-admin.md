# Google Workspace 管理コンソール（admin.google.com）

## 頻繁なアカウント選択画面への差し戻し

`admin.google.com` はセッション中に何度も「Choose an account」画面（`admin.google.com/ac/accountchooser?continue=...`）に差し戻される。`navigate` で目的のURLに遷移したはずが、このアカウント選択画面になっていることがある。

- アカウント一覧から目的のアカウントの `listitem` をクリックしても、1回目のクリックはハイライトするだけで遷移しないことがある。`wait` を挟んでから同じ座標を再度クリックする、または少し `wait` してから画面を再確認する
- `find` で要素を掴んでクリックしても遷移しないことがあるため、遷移したかどうかは毎回スクリーンショットまたは `tabs_context_mcp` のタイトルで確認する

## グループ作成フォーム

- 「グループ名」を入力してもメールアドレス欄は自動生成されない。メールアドレス欄（`@fec.or.jp` のようにドメインがサフィックス表示される）に手動で入力する必要がある
- 外部（組織外）からのメールを受信する用途のグループ（`info@` など問い合わせ受け口）を作る場合、デフォルトの「公開」アクセスタイプでは「投稿できるユーザー」の「外部」列が許可されておらず、外部からのメールを拒否してしまう。「カスタム」に切り替えて「投稿できるユーザー」の「外部」列にチェックを入れる必要がある
- グループ作成ウィザードの「グループのオーナー」欄で追加した人物は、その後の「メンバーを追加」で重複追加しようとすると「既にメンバーになっている」エラーになる（オーナーは自動的にメンバーも兼ねる）

## メンバー一覧の反映遅延

メンバー追加操作の直後、同一画面のメンバー一覧に反映されないことがある（追加時のエラーメッセージでは「既にメンバー」と出るのに一覧には表示されない）。`navigate` で同一URLに再遷移し、数秒 `wait` してから再確認すると正しく反映される。

## 日付ピッカー（適用日の指定など）

「指定日以降に強制」のような日付指定欄のカレンダーピッカーは、目的の日付の `gridcell` を1回クリックしただけでは選択が反映されず、カレンダーが開いたままDate欄が空のことがある。`wait` を1〜2秒挟んでから同じ日付のセルを再度クリックする、または `find` でその時点の `ref` を取り直してから再クリックすると反映される。クリック直後は必ずスクリーンショットで日付欄の表示値を確認し、意図した日付になっているか（前後の日付を誤クリックしていないか）を確認すること。

## セッション切れによるアカウント選択画面への差し戻し（同一URL再遷移時）

`navigate` で同一URLに再遷移するとセッションが切れ、`admin.google.com` がアカウント選択・パスキー確認画面（`accounts.google.com/v3/signin/...`）に差し戻されることがある。この画面での要素クリックはClaude側からは反映されないことが多く、パスキー等の本人確認はユーザー自身の操作が必須。同一URLへの再遷移は避け、ページ内のリンク・メニュー項目（`read_page` で取得した `ref`）経由で遷移すること。

## 「Can't enforce 2-Step Verification」ダイアログ

管理者自身のアカウントで2段階認証を有効化していない状態で組織への2FA適用を保存しようとすると、「Can't enforce 2-Step Verification」ダイアログが出て保存がブロックされる。ダイアログ内の「2 段階認証プロセスを有効にする」ボタンはクリックしても反応しないことがあり、その場合は管理者自身が `myaccount.google.com/signinoptions/twosv` で2段階認証（パスキー登録含む）を先に有効化する必要がある。パスキー登録自体（生体認証やセキュリティキーのタップ）はユーザー本人の操作が必須でClaudeからは代行できない。有効化後は管理コンソールのURLに再度 `navigate` すると設定フォームがリセットされる（未保存の変更が失われる）ため、日付指定などを再入力してから保存し直すこと。

## 新しいユーザーの追加フォーム（`ac/user/bulkadd`）

- 「名」欄に日本語で入力した直後に「メインのメールアドレス」欄をクリックして入力すると、名の入力内容がメールアドレス欄の先頭に混入することがある（例:「真由」+「mayu.sakaguchi」→「真由mayu.sakaguchi」）。メールアドレス欄は入力後に必ずスクリーンショットで値を確認し、混入していたら `cmd+a` → `Delete` で全消去してから入力し直すこと
- 入力完了後、フォームの「続行」ボタンは最初クリックしても無効なまま反応しないことがある。メールアドレス欄の下に「利用可能」という検証結果が表示されてから再度クリックすると進む
- ユーザー作成完了後の画面で「予備のメールアドレス」に個人の連絡先メールを設定していれば、「ログイン手順を送信する」ボタンでそのアドレス宛に初回ログイン情報を送信できる。ここも1回目のクリックで送信済み表示にならないことがあるため、送信結果（「成功しました。ログイン手順を...送信しました」）が表示されるまで確認すること

## グループ一覧からの直接メンバー追加

グループ詳細ページに遷移しなくても、グループ一覧（`ac/groups`）で対象行にカーソルを合わせると行末に「メンバーを追加」等のアクションが表示され、そこから直接メンバー追加ダイアログを開ける。ダイアログはクリック直後は青いプレースホルダー表示のまま読み込み中のことがあるため、`wait` を1秒程度挟んでから入力を始めること。

## 複数アカウントログイン時のmyaccount.google.comのコンテキスト不一致

複数のGoogleアカウントが同時にログインしているブラウザで `myaccount.google.com` に `navigate` すると、目的のアカウントではなく最後にアクティブだった別アカウント（個人のGmailアカウント等）の個人情報ページが開かれることがある。住所・電話番号・誕生日など機微情報が画面に表示されるため注意が必要。

ページ右上のアカウントアイコンをクリックし、アカウント切り替えメニューから目的のアカウントを明示的に選択してから作業を続けること。表示されているメールアドレス（ページ上部やプロフィール情報）が目的のアカウントと一致しているか、作業前に必ず確認する。
