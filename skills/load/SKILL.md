# load

引数: $ARGUMENTS

- `$0`: issue/PR番号（`gh` コマンドの対象）
- `$0` 以降に文字列が続く場合、読み込み後に実行する後続指示として扱う

## 現在の環境

- リポジトリ: !`gh repo view --json nameWithOwner -q .nameWithOwner 2>/dev/null || echo "不明"`
- 現在のブランチ: !`git branch --show-current`
- `#$0` の種別: !`gh api "repos/{owner}/{repo}/issues/$0" --jq 'if .pull_request then "PR" else "issue" end' 2>/dev/null || echo "見つかりません"`

この処理はフォークされた独立コンテキストで実行される。呼び出し元の会話履歴は参照できないため、`$ARGUMENTS` に含まれる情報のみが前提となる。

## 対象が見つからない場合

- 「対象が見つからない」旨と、番号の確認が必要である旨を最終報告として返す（後続処理は行わない）
- フォークしたリポジトリで作業している場合は、親リポジトリのissue/PRが対象である可能性がある旨も報告に含める

## 情報収集

種別ごとに以下の情報を**並列で**収集する。独立したツール呼び出し（`gh` コマンド・`WebFetch` 等）は積極的に並列化すること。

### issueの場合

1. **本文・コメント**: `gh issue view $0 --comments`
2. **言及されている issue / PR**: 本文・コメント中に `#xxx` 形式があれば `gh issue view` / `gh pr view` で内容を確認する（1段階のみ辿る）
3. **対応PRの有無**: `gh pr list --search "#$0" --state all` で、このissueに対応する（言及していない）既存PRがないか確認し、見つかった場合は本文・差分・レビューコメントも取得する
4. **外部リンク**: 外部URLがあれば `WebFetch` で内容を取得する

### PRの場合

1. **本文・コメント**: `gh pr view $0 --comments`
2. **差分**: `gh pr diff $0`
3. **レビューコメント**: `gh pr view $0 --json reviews --jq '.reviews[] | select(.body != "") | "[\(.author.login)] \(.state): \(.body)"'`
4. **CIステータス**: `gh pr checks $0`
5. **言及されている issue / PR**: 本文・コメント・レビューコメント中に `#xxx` 形式があれば `gh issue view` / `gh pr view` で内容を確認する（1段階のみ辿る）
6. **外部リンク**: 外部URLがあれば `WebFetch` で内容を取得する

この処理はフォークされた独立コンテキストで実行されるため、収集した本文・コメント・差分などの生データはこの中でのみ保持され、呼び出し元の会話には残らない。呼び出し元が後から活用できるよう、最終報告には要点を過不足なく含めること。

## 後続指示がない場合

- 最終報告として以下を返す。生データの垂れ流しではなく、後で質問や作業の前提として使える粒度の要約にする。

```
## 読み込み完了: [issue|PR] #<番号> <タイトル>

<本文・コメントの要点、関連issue/PRの要点、現状・論点・懸念点などの簡潔な要約（箇条書き可）>
```

- 実装・ブランチ作成・コメント投稿などのアクションは行わない

## 後続指示がある場合

- 収集した内容を前提として、このフォーク内で後続指示の実行まで行う
- 後続指示はユーザからの直接の依頼として扱う
- 実装系の指示であっても、このフォーク内ではコミット・PR作成までは行わない（`commit` / `pr` スキルに委ねる）
- 最終報告には、実行結果と、呼び出し元が把握しておくべき前提（対象issue/PRの要点）を含める
