# コードを編集する際に守るべきルール

## スコープ管理

- 与えたタスクに関連のない箇所の変更を行うことを禁止します。代わりにそれはユーザに報告してください(ex: 検索中に発見した別のバグやタイポなど)
- 後続のPRで対応するとした箇所については、`TODO: {title} ({issue_url})`の形式でコメントを残すこと

## コメント

- コードを編集する際、変更内容を説明するコメントを絶対に追加せず、必要な説明は会話の中で行う（JSDocなどのドキュメントコメントは除く）

## 既存コードとの整合性

- 実装にあたっては、既存コードとの整合性を重視し、既存に同様の実装がある場合にはそれらに倣って実装を行うこと
- 共通関数やユーティリティを使用する際、該当箇所のニーズを満たすために、無理に共通関数を拡張して使用することは避ける（ex: 本来の責務と異なるオプションを追加する/汎用関数の内部ロジックを安易に変更する）
- 関数や型ファイルを追加/更新する際には、同様の機能を持つ関数などが既に存在しないかコードベースを検索し、必要に応じてそれらを再利用/更新対象にすること

## 関数設計

- 関数は可能な限り純粋関数として実装し、副作用を伴う処理は丁寧に分離すること（ex: 関数内で外部に依存する処理などを記述しないこと）

## 変更時の影響確認

- ファイルの変更・削除を行う際には、ドキュメント内にそれらのファイルへの言及がないか、コードベースを検索し、必要に応じて更新すること
