# JavaScript/TypeScriptを編集する場合に守るべきルール

## JSDoc

- JSDocを必ず記載すること（example等、詳細すぎるものは不要/既存にコメントが存在する場合には書き換えない）
- JSDocは1行だけであっても、必ず複数行で記載すること

## 関数型プログラミング / イミュータビリティ

- 関数型プログラミングを優先：map/filter/reduce を for/while ループより優先
- イミュータブルな操作：スプレッド演算子（...）を使い、push/splice などの破壊的メソッドを避ける
- 配列に対して破壊的な操作(push/pop/etc...)などを禁止し、代わりに非破壊的な操作(map/filter/etc...)を使用すること
- 純粋関数を心がけ、副作用を丁寧に分離すること

## 変数 / 定数

- const を優先し、let/var の使用を禁止します
- マジックナンバーは定数で置き換え、インポート文の下に配置する(ex: const MIN_BOX_SIZE = 50 as const; // 定数の意味を説明するコメント)

## 制御フロー / 可読性

- else if を避け、早期リターンを必ず使用する様にして
- 早期リターンでは、1行で収まる場合、中括弧を省略して記載すること
- 三項演算子が2つ以上ネストする場合、即時関数(IIFE)を使用して可読性を確保すること

## 関数定義

- 関数はfunctionではなくアロー関数で定義すること（既存の関数定義に関しては変更しないこと）

## 非同期処理

- 非同期処理では async/await を優先し、Promiseチェーンを絶対に使用しないこと（既存にPromiseチェーンが存在する場合は変更しないこと）
- 非同期処理を含む処理では、関数全体の設計時点から並列化を目指し、並列で実行可能な箇所では積極的に Promise.all 等を使用して並列化を行うこと

## 文字列

- 文字列を結合するときには、テンプレートリテラルを使用すること

## エラーハンドリング

- catchブロックのエラーログには `error instanceof Error ? error.message : "Unknown error"` を使用し、errorオブジェクトを直接出力しないこと
- 外部APIのエラー詳細はサーバーログに留め、クライアントには汎用メッセージのみ返すこと
